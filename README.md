# Сервис автоматической конвертации нетлистов Altium

Этот микросервис — чистый вайбкодинг. Минимум рук, максимум ИИ (DeepSeek)

Сервис отслеживает появление и изменение файлов нетлистов (`*.net`) в формате Protel Netlist 2.0, создаваемых Altium Designer, и автоматически выполняет три действия:

1. **Упрощённый список соединений** (`*_net.txt`) — только электрические связи, без координат и служебных данных.
2. **Спецификация компонентов** (`*_bom.txt`) — Designator, PartType и Comment для каждого компонента.
3. **Граф соединений для Graphviz** (`*_net.dot`) — визуализация всех цепей в формате DOT, готовая к просмотру в виде схемы.

Сервис работает в фоне как служба Windows, автоматически запускается при старте системы и не требует вмешательства пользователя.

## Логика обработки

Логика обработки (чистки) исходного файла `.net` формата **Protel Netlist 2.0** описана прямо в коде `Worker.cs` и состоит из нескольких этапов.

### Общий алгоритм (`ProcessFile`)

1. **Проверка расширения** – обрабатываются только файлы, оканчивающиеся на `.net` (без учёта регистра).
2. **Задержка 500 мс** – чтобы файл точно освободился после записи его программой Altium.
3. **Создание выходной папки `out`** внутри текущей директории с исходным файлом.
4. **Копирование оригинала** – файл сохраняется как `<имя>_orig.txt`.
5. **Чтение всех строк** и вызов `ParseLines` для извлечения структурированных данных.
6. **Запись трёх выходных файлов**:
   - `_net.txt` – упрощённый список цепей,
   - `_bom.txt` – перечень компонентов,
   - `_net.dot` – граф для Graphviz.

### Детальный разбор `ParseLines`

Парсер двигается строка за строкой и определяет, в какой секции он находится сейчас:

- **Компонентный блок** `[ ... ]`  
  Собирает поля `DESIGNATOR`, `PARTTYPE`, `Comment`.  
  Для этого используется флаг ожидания `expectedField`: когда встречается одно из ключевых слов, следующая непустая строка считается его значением.  
  Все остальные строки (описания, ссылки, `*` и т.д.) игнорируются.  

- **Секция соединений (цепь)** `( ... )`  
  Первая строка после открывающей скобки запоминается как имя цепи (`NetName`).  
  Каждая последующая строка до закрывающей скобки проверяется регулярным выражением `^(?<pin>[A-Za-z0-9]+-\d+)`.  
  Оно извлекает подстроку вида **Буквенно-цифровой\_префикс-Число** (например, `R1-2`, `IC6-15`).  
  Вся остальная информация (комментарии, тип корпуса, `Passive` и т.п.) отсекается.  
  Собранные пины добавляются в цепь, и она считается «готовой», когда встречается `)`.

- **Всё, что находится вне этих блоков** (заголовок `PROTEL NETLIST 2.0`, пустые строки и пр.), игнорируется.

Таким образом, «чистка» заключается в:
- **Удалении** координат, описаний, футпринтов, ссылок на даташиты и прочей служебной информации.
- **Сохранении только электрических связей** – имени цепи и обозначений выводов компонентов.
- **Выделении спецификации (BOM)** с минимальным набором полей (обозначение, тип, комментарий).

Логика сознательно не изменяет содержимое имён цепей или обозначений, а только фильтрует строки, чтобы оставить суть схемы — «кто к кому подключён».

## Функциональность

- **Мониторинг в реальном времени:** `FileSystemWatcher` подписан на события создания, изменения и переименования файлов в указанных папках.
- **Фильтр по расширению:** обрабатываются только файлы с расширением `.net` (регистронезависимо).
- **Автоопределение кодировки:** читает исходный файл, предполагая системную кодировку, а результат всегда сохраняет в UTF-8 без BOM.
- **Упрощение нетлиста:** из секций цепей `( ... )` извлекаются имена цепей и пины (формат `Обозначение-НомерВывода`). Координаты `(X Y)`, служебные строки и заголовок игнорируются.
- **Генерация BOM:** из компонентных блоков `[ ... ]` собираются поля `DESIGNATOR`, `PARTTYPE` и `Comment`, формируя перечень компонентов.
- **Экспорт в Graphviz DOT:** для каждого нетлиста создаётся файл `*_net.dot` с описанием графа «компонент – цепь», что позволяет быстро получить визуальную схему всех соединений.
- **Первичное сканирование:** при запуске сервис обрабатывает все уже существующие `.net` файлы, чтобы не пропустить старые данные.
- **«Горячая» перезагрузка конфигурации:** если в `appsettings.json` добавить или изменить пути к папкам, сервис перестроит наблюдателей без перезапуска.
- **Работа в фоне:** сервис устанавливается как стандартная служба Windows и может запускаться автоматически при загрузке системы.


## Конфигурация

Все настройки хранятся в `appsettings.json`. Пример:

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.Hosting.Lifetime": "Information"
    }
  },
  "WatcherSettings": {
    "SourceDirectories": [
      "D:\\Projects\\Altium\\Generator\\PCB_Project\\Project Outputs for PCB_Project\\Protel2Netlist",
      "C:\\AnotherFolder"
    ]
  }
}
```

- **SourceDirectories** — массив строк с полными путями к папкам, которые нужно мониторить.
- Можно использовать как обратные (`\\`), так и прямые (`/`) слэши.
- Пути должны существовать на момент запуска сервиса. Если папка отсутствует, она будет пропущена с записью в лог.

При изменении этого файла (после публикации службы) сервис автоматически перечитает его и обновит список наблюдаемых папок.

## Сборка и публикация

**Отладочная сборка** (проверка ошибок компиляции):

```bash
dotnet build
```

**Финальная публикация** (оптимизированная версия для установки службы):

```bash
dotnet publish NetFileConverter.csproj -c Release -o C:\Services\NetConverter
```

- `-c Release` — включает оптимизацию кода.
- `-o` — указывает выходную папку, куда будут скопированы `.exe`, `.dll` и файлы конфигурации.

После выполнения в `C:\Services\NetConverter` появятся все файлы, необходимые для работы службы.

## Тестовые запуски

Перед установкой как службы удобно запустить программу в интерактивном режиме.

**Обычный запуск:**

```bash
dotnet run
```

Консоль останется открытой, и вы увидите логи работы. Для остановки нажмите `Ctrl+C`.

**Автоматический перезапуск при изменении кода (watch mode):**

```bash
dotnet watch run
```

Теперь при любом сохранении файлов проект будет пересобираться и перезапускаться автоматически.

**Проверка работы:**
1. Создайте или скопируйте любой файл с расширением `.net` в одну из отслеживаемых папок.
2. В консоли должно появиться сообщение `Обработан: <имя> -> out/<имя>_net.txt + out/<имя>_bom.txt + out/<имя>_net.dot`.
3. В подпапке `out` появятся три файла:
   - `<имя>_net.txt` – список соединений.
   - `<имя>_bom.txt` – спецификация компонентов.
   - `<имя>_net.dot` – граф для Graphviz.

## Пошаговая отладка

VS Code с расширением C# Dev Kit предоставляет полноценный отладчик.

1. Откройте файл `Worker.cs` и установите точку останова (клик слева от номера строки в методе `ProcessFile` или `StartWatchers`).
2. В левом меню выберите **Run and Debug** (или нажмите `Ctrl+Shift+D`).
3. В выпадающем списке конфигураций выберите **.NET Core Launch (console)** (или создайте её, если отсутствует).
4. Нажмите зелёный треугольник (F5) для старта отладки.
5. Когда сработает событие файловой системы, выполнение приостановится на точке останова. Вы сможете инспектировать переменные, выполнять шаги (F10, F11) и анализировать логику работы.

Для отладки событий, происходящих при старте (например, `InitialScan`), просто запускайте отладку — код выполнится по шагам.

## Установка как службы Windows

После успешной публикации сервис можно зарегистрировать в системе.

1. Откройте **PowerShell от имени администратора**.
2. Создайте службу:

```powershell
New-Service -Name "AltiumNetConverter" `
            -BinaryPathName "C:\Services\NetConverter\NetFileConverter.exe" `
            -DisplayName "Altium .NET to .TXT Converter" `
            -StartupType Automatic
```

3. Запустите службу:

```powershell
Start-Service -Name "AltiumNetConverter"
```

4. Проверьте статус:

```powershell
Get-Service -Name "AltiumNetConverter"
```

Статус должен быть `Running`.

**Настройка прав доступа (важно):**

По умолчанию служба работает от имени учётной записи `Local System`, что может ограничить доступ к сетевым дискам или папкам пользователя.

- Нажмите `Win + R`, введите `services.msc` и нажмите Enter.
- Найдите службу **Altium .NET to .TXT Converter**.
- Правой кнопкой → **Свойства** → вкладка **Вход в систему**.
- Выберите **С учётной записью** и укажите логин/пароль вашего пользователя Windows, у которого есть полные права на папки с проектами Altium.
- Нажмите **ОК** и перезапустите службу.

Теперь сервис будет работать в фоне, а логи появятся в **Просмотре событий** (Event Viewer) → «Журналы приложений и служб» → «Приложение».

## Переустановка службы

При изменении кода или конфигурации достаточно повторить шаги:

1. Остановите и удалите службу (как описано выше).
2. Пересоберите проект с новыми изменениями:

```bash
dotnet publish NetFileConverter.csproj -c Release -o C:\Services\NetConverter
```

3. Снова создайте и запустите службу:

```powershell
New-Service -Name "AltiumNetConverter" `
            -BinaryPathName "C:\Services\NetConverter\NetFileConverter.exe" `
            -DisplayName "Altium .NET to .TXT Converter" `
            -StartupType Automatic
Start-Service -Name "AltiumNetConverter"
```

Если вносились только изменения в `appsettings.json` (добавлены/удалены папки), переустанавливать службу не нужно — она сама перечитает конфиг на лету.

## Визуализация соединений с помощью Graphviz

Файл `*_net.dot` можно преобразовать в изображение с помощью утилиты `dot` из пакета Graphviz.

1. **Установите Graphviz**  
   Скачайте установщик с [graphviz.org/download](https://graphviz.org/download/) и установите.  
   Убедитесь, что `dot` добавлен в переменную окружения `PATH`.

2. **Сгенерируйте изображение**  
   Откройте терминал в папке с `.dot` файлом и выполните:

   ```bash
   dot -Tsvg -O PCB1_net.dot
   ```

   Будут созданы файлы `PCB1_net.dot.svg` (векторный SVG). Поддерживаются и другие форматы: `-Tpng`, `-Tpdf`.

3. **Пример графа для вашего проекта**  
   Прямоугольники — компоненты (обозначение и тип), овалы — цепи (имя). Линии показывают, к каким выводам компонентов подключена цепь.

Если схема кажется перегруженной, можно настроить раскладку, добавив в начало `.dot`-файла параметры:

```dot
graph Netlist {
    rankdir=TB;       // вертикальная ориентация
    splines=ortho;    // ортогональные линии
    nodesep=0.5;
    ranksep=1.0;
    ...
}
```

Подробнее о параметрах — в [документации Graphviz](https://graphviz.org/documentation/).

## Возможные проблемы и решения

1. **Сервис не реагирует на новые файлы `.net`**  
   - Убедитесь, что расширение действительно `.net` (служба нечувствительна к регистру, но проверьте маску `FileSystemWatcher`).
   - Проверьте права учётной записи службы на целевую папку. Лучше всего запускать службу от имени конкретного пользователя.
   - Иногда Altium сохраняет файлы через временное переименование — у нас подписано событие `Renamed`, поэтому должно срабатывать.

2. **Ошибка доступа к файлу (IOException)**  
   - Увеличьте задержку `Thread.Sleep` в `ProcessFile`. По умолчанию стоит 500 мс. Для больших проектов можно увеличить до 1000–2000 мс.

3. **Служба не стартует и сразу останавливается**  
   - Откройте «Просмотр событий» → «Приложение» и найдите ошибки с источником `.NET Runtime` или `AltiumNetConverter`.
   - Частая причина – отсутствие целевых папок в `appsettings.json` или опечатка в пути.
   - Попробуйте запустить `.exe` напрямую из командной строки (не как службу), чтобы увидеть ошибки в консоли.

4. **Пустой или неполный `_bom.txt`**  
   - Убедитесь, что в исходном нетлисте в блоках `[ ... ]` присутствуют поля `DESIGNATOR`, `PARTTYPE` и (опционально) `Comment`.
   - Проверьте, что парсер корректно находит эти поля. При необходимости можно расширить список извлекаемых полей в `ParseComponentLine`.

5. **Граф DOT выглядит запутанно**  
   - Попробуйте изменить `rankdir` на `TB` (сверху вниз) или `LR` (слева направо).
   - Включите `splines=ortho` для прямых углов.
   - Для очень больших схем можно скрыть мелкие цепи, добавив фильтр по числу пинов (в коде).

6. **После публикации конфиг не применяется**  
   - Проверьте, что файл `appsettings.json` скопирован в выходную папку. По умолчанию `dotnet publish` копирует его автоматически.
   - Если вы вносили изменения **после** публикации, редактируйте файл именно в `C:\Services\NetConverter`, а не в исходной папке проекта.
