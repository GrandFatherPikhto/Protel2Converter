# Автоматический конвертер Protel2 netlists

## NetFileConverter — Сервис автоматической конвертации файлов

NetFileConverter — это легковесное Windows-приложение на базе .NET 8, сочетающее в себе мощь фоновой службы (BackgroundService) и удобство графического интерфейса Windows Forms. Приложение работает в системном трее, отслеживает изменения в выбранных папках в реальном времени и автоматически обрабатывает новые файлы.
------------------------------
## 🚀 1. Функционал программы

* Фоновый мониторинг (File Watcher): Непрерывное отслеживание добавления и изменения файлов в нескольких директориях одновременно с помощью FileSystemWatcher.
* Интеграция в системный трей: Приложение не занимает место на панели задач, полностью работая из трея.
* Динамическая конфигурация: Настройки папок хранятся в стандартном файле appsettings.json. Приложение на лету подхватывает изменения без необходимости ручного перезапуска службы.
* Уведомления Windows: Всплывающие подсказки (Balloon Tips) оперативно информируют пользователя об успешной конвертации или возникших ошибках.
* Полноценный инсталлятор: Автоматическое развертывание, прописка в системный автозапуск для всех пользователей и чистое удаление через Панель управления Windows.

### Функциональность

- **Мониторинг в реальном времени:** `FileSystemWatcher` подписан на события создания, изменения и переименования файлов в указанных папках.
- **Фильтр по расширению:** обрабатываются только файлы с расширением `.net` (регистронезависимо).
- **Автоопределение кодировки:** читает исходный файл, предполагая системную кодировку, а результат всегда сохраняет в UTF-8 без BOM.
- **Упрощение нетлиста:** из секций цепей `( ... )` извлекаются имена цепей и пины (формат `Обозначение-НомерВывода`). Координаты `(X Y)`, служебные строки и заголовок игнорируются.
- **Генерация BOM:** из компонентных блоков `[ ... ]` собираются поля `DESIGNATOR`, `PARTTYPE` и `Comment`, формируя перечень компонентов.
- **Экспорт в Graphviz DOT:** для каждого нетлиста создаётся файл `*_net.dot` с описанием графа «компонент – цепь», что позволяет быстро получить визуальную схему всех соединений.
- **Первичное сканирование:** при запуске сервис обрабатывает все уже существующие `.net` файлы, чтобы не пропустить старые данные.


### Требования

- **Windows 10/11** или **Windows Server 2016+** (для работы в качестве службы)
- **.NET SDK 8.0** или новее (проект совместим с .NET 10)
- **VS Code** (рекомендуется) с расширениями:
  - C# Dev Kit (Microsoft)
  - Error Lens
  - NuGet Gallery
- **Graphviz** (опционально, для визуализации DOT-файлов). Скачать можно с [graphviz.org](https://graphviz.org/download/).

## 💻 2. Как с ней работать## Запуск и управление в трее

   1. После установки программа автоматически запускается и сворачивается в системный трей (около часов).
   2. Нажмите правой кнопкой мыши по иконке приложения в трее, чтобы открыть контекстное меню:
   * Настройки — открывает окно управления отслеживаемыми папками.
      * Выход — полностью закрывает приложение и останавливает фоновую службу.
   3. Двойной клик по иконке в трее также открывает окно настроек.

## Настройка папок для отслеживания

   1. Откройте окно «Настройки».
   2. В появившемся окне отображается текущий список сканируемых директорий.
   3. Нажмите «Добавить папку» и выберите нужный каталог на диске через стандартный диалог Windows.
   4. Чтобы удалить папку из мониторинга, выберите её в списке и нажмите «Удалить».
   5. Нажмите «Сохранить». Настройки запишутся в файл appsettings.json, и служба автоматически переинициализирует наблюдателей в течение пары секунд.

## 🛠 3. Разработка: Очистка, Сборка и Публикация
Управление проектом и сборка осуществляются через терминал VS Code с помощью CLI-команд .NET.
## Очистка проекта
Если необходимо удалить старые артефакты компиляции и очистить кэш сборки:

dotnet clean

## Тестовая сборка (Debug)
Для проверки кода на наличие ошибок компиляции и предупреждений:

dotnet build

## Публикация в один файл (Publish Release)
Для создания релизной версии приложения используется режим Single File. Программа упаковывает все системные .dll зависимости фреймворка .NET 8 внутрь одного исполняемого файла, что позволяет запускать его на компьютерах без установленного .NET Runtime.
Выполните команду публикации:

dotnet publish -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true /p:PublishReadyToRun=true /p:IncludeNativeLibrariesForSelfExtract=true

Результат публикации появится в директории:
bin\Release\net8.0-windows\win-x64\publish\
В этой папке будут находиться всего два файла, необходимые для работы программы:

   1. NetFileConverter.exe — само приложение (со встроенной иконкой).
   2. appsettings.json — конфигурационный файл настроек.

## 📦 4. Создание инсталлятора
Для создания профессионального установщика NetFileConverter_Setup.exe используется бесплатная утилита Inno Setup. Скрипт сборки уже полностью настроен, решает проблемы с правами доступа администратора (admin) и исключает появление предупреждений компилятора.
## Скрипт инсталлятора (installer.iss)
Конфигурационный файл installer.iss находится в корневой папке проекта. Его ключевые особенности:

* Автозапуск: Создает общесистемный ярлык в папке {commonstartup}. Программа автоматически стартует в трее при включении ПК для всех пользователей.
* Ярлык на Рабочем столе: Создает удобную иконку запуска через {commondesktop}.
* Безопасное обновление: В секции [Code] прописан автоматический вызов taskkill для закрытия старой версии программы перед установкой или удалением, чтобы файлы не блокировались ОС.
* Сохранение настроек: Благодаря флагу onlyifdoesntexist, при обновлении программы поверх старой версии файл appsettings.json с путями пользователя не перезаписывается.

## Как собрать инсталлятор:

   1. Убедитесь, что вы выполнили команду dotnet publish (описанную в шаге 3).
   2. Откройте файл installer.iss через программу Inno Setup Compiler.
   3. Нажмите комбинацию клавиш Ctrl + F9 (или выберите пункт меню Build -> Compile).
   4. Процесс компиляции займет около 20-25 секунд.

Готовый к распространению файл установщика будет сохранен в папку:
📁 .\installer_output\NetFileConverter_Setup.exe
