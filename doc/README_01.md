## Проект: Altium .NET to .TXT Auto-Converter Service## 1. Задача проекта
Создание фоновой службы Windows, которая в реальном времени мониторит заданные рабочие директории Altium Designer. При появлении или обновлении файлов с расширением *.NET, сервис автоматически конвертирует их в формат *.TXT, меняет кодировку на UTF-8 и сохраняет результат в подпапку txt.
------------------------------
## 2. Настройка рабочего места (VS Code)## 2.1. Установка .NET
Для работы требуется инструментарий разработчика .NET SDK.

* Рекомендуемая версия: .NET 8.0 (LTS) или выше.
* Ссылка: [Скачать .NET SDK](https://dotnet.microsoft.com/download)
* Команда проверки: dotnet --version

## 2.2. Настройка VS Code
В разделе Extensions (Ctrl+Shift+X) были установлены следующие плагины:

   1. C# Dev Kit — основной пакет для разработки на C#.
   2. Error Lens — подсвечивает ошибки прямо в строках кода.
   3. NuGet Gallery — удобный интерфейс для поиска и установки библиотек.

## 2.3. Установка библиотек (NuGet)
Для превращения консольного приложения в системную службу использовались следующие пакеты (устанавливаются в папке проекта):

dotnet add package Microsoft.Extensions.Hosting.WindowsServices
dotnet add package Microsoft.Extensions.Configuration.Json

------------------------------
## 3. Логика сервиса (Инфраструктура)
Проект построен на шаблоне Worker Service:

* FileSystemWatcher: Основной механизм слежения. Настроен на события Created, Changed и Renamed с фильтром *.*.
* Hot Reload: Сервис использует IConfiguration с флагом reloadOnChange: true, что позволяет добавлять новые папки в appsettings.json без перезагрузки самой службы.
* Initial Scan: При старте сервис не только ждет новых событий, но и сканирует папки на наличие уже существующих файлов.

------------------------------
## 4. Логика программы (Алгоритм)
Метод обработки файлов работает по следующим правилам:

   1. Фильтрация: Проверка расширения файла на соответствие .net (регистронезависимо).
   2. Безопасность: Пауза Thread.Sleep(500), чтобы дождаться завершения записи файла сторонним софтом (Altium).
   3. Организация: Автоматическое создание подпапки txt в директории источника.
   4. Трансформация: Чтение содержимого и принудительная запись в файл с тем же именем, но расширением .txt в кодировке UTF-8.

------------------------------
## 5. Сборка и отладка

* Запуск в режиме разработки:

dotnet run

* Режим авто-перезагрузки при правке кода:

dotnet watch run

* Финальная сборка (Publish):

dotnet publish -c Release -o C:\Services\NetConverter


------------------------------
## 6. Установка сервиса в Windows## 6.1. Размещение и Конфигурация
После публикации сервис становится автономным.

* Путь к исполняемому файлу: C:\Services\NetConverter\NetFileConverter.exe
* Рабочий конфиг: C:\Services\NetConverter\appsettings.json (именно здесь теперь нужно менять пути к папкам Altium).

## 6.2. Регистрация и Запуск
Выполняется в PowerShell от имени Администратора:

   1. Создание службы:
   
   New-Service -Name "AltiumNetConverter" -BinaryPathName "C:\Services\NetConverter\NetFileConverter.exe" -DisplayName "Altium .NET to .TXT Converter" -StartupType Automatic
   
   2. Запуск службы:
   
   Start-Service -Name "AltiumNetConverter"
   
   3. Настройка прав (опционально): Через services.msc -> Свойства -> Вход в систему -> Указать данные пользователя для доступа к дискам D:, E: и т.д.
